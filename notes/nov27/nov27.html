1. A DSL for probabilistic programming

Let's start putting together a little domain-specific language (DSL) for constructing probabilistic programs. To do this, let's first introduce a new atomic type, \(r\), representing the type of real numbers, in addition to \(e\) and \(t\). Second, let's introduce a new type constructor \(\mathtt{P}\), which takes any type \(Œ±\) onto the type \(\mathtt{P} Œ±\) of probabilistic programs which compute things of type \(Œ±\). That is, if \(m : \mathtt{P} Œ±\), then m encode a probability distribution over things of type \(Œ±\).

Our DSL may be associated with the following terms, which we'll describe in turn: \[return : Œ± ‚Üí \mathtt{P} Œ±\] \[(‚àº) : \mathtt{P} Œ± ‚Üí (Œ± ‚Üí \mathtt{P} Œ≤) ‚Üí \mathtt{P} Œ≤\] \[factor : r ‚Üí \mathtt{P} ‚ãÑ\] \[ùîº_{(¬∑)} : \mathtt{P} Œ± ‚Üí (Œ± ‚Üí r) ‚Üí r\] \[ùüô : t ‚Üí r\]

1.1. \(return\)

\(return\) is used to create trivial probabilistic programs; that is, ones which correspond to distributions assigning all of their mass to the returned value (a.k.a. degenerate distributions). For example \(return(j)\) is the program of type \(\mathtt{P} e\) which has a probability of 1 of returning \(j\) and a probability of 0 of returning any \(x\) of type \(e\) which is not \(j\).

1.2. \((‚àº)\)

\((‚àº)\) (pronounced 'bind') lets you sample a value from one probabilistic program (of type \(\mathtt{P} Œ±\)) and feed it into another probabilistic program that depends on a value of type \(Œ±\) (i.e., a function of type \(Œ± ‚Üí \mathtt{P}\)), in order to get back a new probabilistic program of type \(\mathtt{P} Œ≤\). In general, we will write binds in the form \[\begin{array}{l} \mbox{} x ‚àº m \\ k(x) \end{array}\] instead of \(m ‚àº Œªx.k(x)\), in order to suggest an imperative metaphor whereby \(x\) is sampled from \(m\) before the program continues with \(k(x)\).

1.3. \(factor\)

Given a real number \(r\), \(factor(x)\) weights its continuation by \(x\). That is, \[\begin{array}{l} x ‚àº m \\ factor(r) \\ k(x) \end{array}\] is the program which samples \(x\) from \(m\) and then weights by \(r\) the probability of continuing the program with \(k(x)\). For example, if some value \(y\) were to then be sampled from \(k(x)\), the probability associated with \(y\) will now be scaled by \(r\).

1.4. \(ùîº_{(¬∑)}\)

\(ùîº_{(¬∑)}\) is the expected value operator. Given a probabilistic program \(m\) of type \(\mathtt{P} Œ±\) and a function \(f\) mapping any given \(Œ±\) to a real number (\(f : Œ± ‚Üí r\)), \(ùîº_m(f)\) is the expected value of \(f\), given the distribution represented by \(m\). For example, \(m\) might be a distribution over people, and \(f\) might be a function mapping each person to their height. In that case, \(ùîº_m(f)\) is a weighted average of heights of people in the distribution \(m\).

1.5. \(ùüô\)

\(ùüô\) is the indicator function mapping inhabitants of \(t\) to \(r\) (i.e., \(ùüô : t ‚Üí r\)). If \(t\) is \(Bool\), we'd have \(ùüô(\mathtt{T}) = 1\) and \(ùüô(\mathtt{F}) = 0\).
